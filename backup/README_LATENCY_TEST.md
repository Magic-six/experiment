# 延迟对比实验

此实验旨在比较协议在不同网络延迟条件下的性能表现，特别是局域网(LAN)和广域网(WAN)环境下，评估通信量优势如何转化为总运行时间的优势。

## 实验背景

在安全计算协议中，网络延迟对整体性能有显著影响。尤其是在广域网(WAN)环境中，通信延迟可能成为主要瓶颈。此实验通过在不同网络条件下测试Lagrange插值协议，验证其通信优化策略在高延迟环境中的效果。

## 实验设置

实验模拟了以下网络环境：

1. **局域网环境**:
   - LAN (10ms延迟)
   - LAN (50ms延迟)

2. **广域网环境**:
   - WAN (50ms延迟)
   - WAN (100ms延迟)
   - WAN (200ms延迟)

对每种网络环境，实验使用不同数量的参与方(3, 5, 7, 9)进行测试，每个配置重复运行多次以确保结果可靠。

## 运行实验

执行以下命令运行实验：

```bash
python run_latency_comparison.py
```

## 实验结果

实验结果将保存在`network_test_results`目录下，包括：

1. **原始数据**: `latency_comparison_results.json`
2. **图表分析**:
   - `latency_runtime_by_parties.png`: 不同参与方数量下各网络环境的运行时间
   - `latency_runtime_vs_datasize.png`: 运行时间与数据量关系
   - `latency_lan_vs_wan_comparison.png`: LAN与WAN环境的直接对比
   - `latency_compute_vs_network_time.png`: 计算时间与通信时间占比分析

## 分析重点

实验分析重点关注：

1. **通信量优势转化**: 验证协议的通信优化如何在高延迟环境中提供性能优势
2. **计算与通信占比**: 分析在不同网络条件下，计算时间与通信时间的比例变化
3. **可扩展性**: 评估随着参与方数量增加，性能如何变化
